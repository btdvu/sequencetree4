## 1. Waveform Output (`sim.dat` & `sim_filtered.dat`)
This format represents a serialized [STSimScannerBlockList](file:///simulator/stsimscannerblocklist.h#L100). The byte alignment matches C++ data sizes (without padding).

### Binary Layout
1. **Global Header**:
   * `max_RF_amplitude` (`double` - 8 bytes)
   * `num_blocks` (`long` - 8 bytes)
2. **Blocks** (repeated `num_blocks` times):
   * `m_duration` (`double` - 8 bytes)
   * `m_block_number` (`long` - 8 bytes)
   * `m_elapsed_time` (`double` - 8 bytes)
   * `m_wait_for_trigger` (`bool` - 1 byte)
   * `num_events` (`long` - 8 bytes)
3. **Events** (repeated `num_events` times for each block):
   * **Event Header** (60 bytes):
     * `block_index`, `tree_index`, `event_type` (`int` x 3 - 12 bytes total; type 1=RF, 2=Readout, 3=Gradient, 4=ArbGradient).
     * `start_time`, `duration`, `reference_time`, `timestep`, `phase`, `frequency` (`double` x 6 - 48 bytes total).
   * **RF Segment**:
     * `num_rf_points` (`long` - 8 bytes) + mag/phase data (`double` x 2 x `num_rf_points`).
   * **Readout Segment** (65 bytes):
     * `num_points` (`long` - 8 bytes) + `onlineADC` (`bool` - 1 byte) + 14 index fields (`int` x 14 - 56 bytes).
   * **Gradient Segment**:
     * `direction` (`int` - 4 bytes) + `num_times` (`long` - 8 bytes) + time/amp data (`double` x 2 x `num_times`).

### Python Waveform Parser

See [`parse_simulation_dat()`](file:///home/brian/sequencetree4/agentic_coding/parse_sim_dat.py) in the canonical module.
The function returns a list of block dicts, each containing an `events` list with fully-decoded fields:

| Field | Description |
|---|---|
| `event_type_name` | `"RF"`, `"Readout"`, `"Gradient"`, or `"ArbGradient"` |
| `start_time` | Event start time within the TR block (µs) |
| `duration` | Total event duration (µs) |
| `reference_time` | Reference (e.g. echo center) time (µs) |
| `timestep` | Waveform sample interval (µs) |
| `rf_points` | List of `(mag, phase)` tuples (uT, deg) — non-empty for RF events |
| `readout_num_points` | ADC sample count — non-zero for Readout events |
| `readout_online_adc` | Whether the ADC was armed online |
| `grad_axis` | `"X"`, `"Y"`, or `"Z"` — set for Gradient/ArbGradient events |
| `grad_times` | List of breakpoint times (µs) |
| `grad_amps` | List of gradient amplitudes (uT/mm) |

## 2. Hardware Template Output (`sim_raw.dat`)
This format logs sequence steps and loop increments rather than physical shapes. It consists of consecutive 4-byte integer code packets:

* **Code `5` (Readout Event)**: `adc_index` (`int`), `N` (`int`)
* **Code `10`/`15`/`20` (Iterator ops)**: `iterator_index` (`int`). Codes correspond to Init (10), Step (15), and Reset (20).
* **Code `25` (Set Readout Index)**: `num_bytes` (`int` = 12), `adc_index` (`int`), `iter_index` (`int`), `value` (`int`)

### Python Template Parser

See [`parse_raw_template()`](file:///home/brian/sequencetree4/agentic_coding/parse_sim_dat.py) in the canonical module.
The function returns a list of command dicts keyed by `code`:

| Code | Fields | Meaning |
|---|---|---|
| `5` | `adc_index`, `num_points` | ADC readout event |
| `10` | `op="Init"`, `iterator_index` | Iterator initialised |
| `15` | `op="Step"`, `iterator_index` | Iterator incremented |
| `20` | `op="Reset"`, `iterator_index` | Iterator reset |
| `25` | `adc_index`, `iterator_index`, `value` | Readout k-space index assigned |
